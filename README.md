# MitM Collector: CSV / Excel (File Upload)

This directory contains the autonomous Go binary for parsing, encrypting, and ingesting manual CSV (and optionally XLSX) file uploads into the MitM Data Aggregator ecosystem.

## Architecture & Triggering

Unlike typical scheduled database collectors (e.g. `mitm_collector_pg`), this collector is **triggered dynamically** by the `mitm_scheduler` upon receiving a file upload through the REST API (`POST /admin/upload/source_file`).

1. A user with the `UPLOADER` or `ADMIN` role uploads a file via the MitM Admin Frontend.
2. The Go Scheduler temporarily stores the file in the configured `upload_dir`.
3. The Scheduler immediately launches this collector using its internal job runner engine.
4. The collector parses the CSV, encrypts it on a row-by-row basis (generating its own Data Encryption Key).
5. It extracts a deterministic `correlation_id` based on the specified `business_key_column` and inserts the fragments into the `raw_ingestion` table.
5. IPC logs are actively streamed back to the Scheduler.
6. The temporary file is securely removed (`os.Remove`) from the filesystem upon completion or failure.

## Required Parameters

The Collector receives its dynamic arguments via the first command-line argument (`os.Args[1]`) as a JSON string:

```json
{
  "file": "/path/to/uploaded/file.csv",
  "topic": "TARGET_TOPIC_NAME",
  "business_key_column": "Personalnummer"
}
```

* `business_key_column`: (Optional) The CSV header column to use for generating a deterministic `correlation_id` (UUIDv5). If omitted, the first column of the CSV is used as a fallback. This allows linking manual uploads with database ingestion events.

## Required Environment Variables

The Collector relies on the following environment variables (which are automatically injected by the `mitm_scheduler` engine):

| Variable | Description | Example |
| :--- | :--- | :--- |
| `MASTER_KEY` | The KEK (Key Encryption Key) used for Envelope Encryption. Must be a 32-byte Base64 string or 32-byte raw string. | `o3+A5...=` |
| `RUN_ID` | The numeric Job Run ID generated by the Scheduler. Used for assigning logs to the correct job run. | `1531` |
| `SCHEDULER_SOCKET_PATH` | UNIX Domain Socket path for IPC communication with the Scheduler. | `/tmp/scheduler.sock` |
| `MITM_DB_CONFIG_JSON` | (**Preferred**) JSON-encoded credentials containing a nested `"db"` object for the MitM PostgreSQL database. | `'{"db": {"host":"localhost","port":5432,...}}'` |
| `MITM_DB_HOST` (and others) | (**Fallback**) Direct environment variables for the PostgreSQL connection if JSON is missing. | `192.168.0.31` |

## Compilation

The `mitm_collector_csv_xls` is designed to be completely standalone.

```bash
cd collector-layer/mitm_collector_csv-xls
go build -o ../../bin/mitm-collector-csv-xls main.go
```

## Security

* Requires the `MASTER_KEY` environment variable for Envelope Encryption (automatically passed by the scheduler).
* Temporarily stored plaintext data is purged from the host operating system instantly after ingestion.
